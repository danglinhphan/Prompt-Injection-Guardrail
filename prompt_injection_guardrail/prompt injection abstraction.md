Prompt injection attacks pose a critical threat to Large Language Model (LLM)-powered applications, including chatbots and Retrieval-Augmented Generation (RAG) systems. The Open Worldwide Application Security Project (OWASP) Top 10 for LLM Applications identifies this vulnerability class as a leading risk, encompassing direct injection, indirect injection via retrieved documents, data exfiltration, and misuse of tools. These attacks can result in policy violations, unauthorised data access, and manipulation of model outputs. This work proposes a lightweight guardrail middleware that operates as a protective layer before LLM inference, performing three core functions: (1) multi-class prompt classification into five categories (benign, injection-direct, injection-indirect, data-exfiltration, tool-misuse) derived from the OWASP Prompt Injection Cheat Sheet; (2) policy-driven action recommendation that maps predicted labels and confidence scores to actionable decisions (allow, warn, block, sanitize), with context-aware mitigations such as stripping instruction overrides and disabling tool calls; and (3) explainable risk attribution at the token level using SHapley Additive exPlanations (SHAP) for Term Frequency - Inverse Document Frequency (TF-IDF) baselines and Integrated Gradients for neural models, highlighting the specific prompt segments that triggered the classification. Two model tiers are evaluated: TF-IDF with Logistic Regression and DistilBERT fine-tuned with Low-Rank Adaptation (LoRA) for parameter-efficient training. Robustness is assessed through paraphrasing, multilingual (Vietnamese/English), and noise perturbation experiments. The system is deployed as a FastAPI endpoint with a Gradio demo on Hugging Face Spaces. The entire pipeline, from data curation to deployment, is designed to be fully reproducible within 15 days at near-zero cost, demonstrating that effective prompt-injection defence is accessible without requiring large-scale infrastructure.
